Areas de GIT

Working directory

El area de trabajo corresponde al a carpeta donde inicializamos el repositorio de git, rara vez vamos a estar utilizando la linea de comandos y es donde basicamente vamos a crear, editar y eliminar los diferentas archivos de nuestro proyecto

  • dir || ls muestra el contenido de la carpeta actual
  • mkdir <name crea una carpeta con el nombre especificado
  • cd <path cambia la carpeta actual por la seleccionada
  • git init inicializa un repositorio de git en al carpeta actual
  • rm -rf .git elimina el repositorio de manera forzada

Staging Area (INDEX)

El area de control de cambios es aquella donde agregamos los archivos a los que vamos a estar haciendoles seguimiento, cada vez que agregamos un elemento al index, se realiza una captura de pantalla del codigo (SNAPSHOT) que luego debera ser confirmada para determinar su permanencia en el historial de cambios.

  • git status: muestra el estado de los archivos del working directory con respecto al INDEX, dentro de los estamos mas comunes estan:
    • untracked: no se esta realizando el seguimiento del archivo, irrecuperable en el caso de perdida
    • staged: se realizo la captura del codigo pero todavia no se confirmaron los cambios
    • commited: los cambios realizados en el archivo se encuetran almacenados en el historial
    • modified: el archivo difiere con respecto a lo registrado en el INDEX o la ultima confirmacion
  • git add <file agrega al INDEX el conjunto de archivos seleccionados
  • git diff compara los archivos del Working Directory con lo que se registra en el INDEX.
  • git restore --staged <file elimina la ultima captura del INDEX, no modifica el archivo
  • git rm --cached <file elimina el archivo INDEX

Repository (LOCAL/REMOTE)

es el area donde se almacenan todas las confirmaciones de cambios en archivos de tipo BLOB(Binary Large Object File) distribuidos en una estructura de carpetas identifdicadas de manera hexadecimal, que van desde el 00 al ff. Esto nos permite crear punto de control a los que podemos volver en cualquier momento

  • git commit abre el editor predeterminado para realizar la confirmacion de cambios
  • git log muestra el historial de cambios ordenados de manera descendente
  • git blame <file muestra los autores y confirmaciones de cada linea en un archivo
  • git push <remote <branch agrega los cambios de la rama al repositorio remoto
  • git fetch <remote captura los cambios del repositorio remoto
